# 3D 导演台 · 技术改动规划（第二轮）

> 范围：本轮聚焦三块——**(A) 底部 Dock 图标微调**、**(B) 机位视角 POV 渲染 + 右侧「摄像机」属性面板**、**(C) 截图数据模型与「摄像机截图」管理**。
> 衔接第一轮：摄像机已是一等实体 `CameraEntity`（`type:'camera'`，含真实 `cam` / 相机体 / `CameraHelper` / 名牌）。本轮把第一轮标为「二期可选」的 **POV 渲染正式纳入**。
> 已确认的关键决策（来自需求澄清）：
> 1. **机位视角 = POV 真实渲染**：主视口与预览图都透过所选相机出画面，截图按该相机出图。
> 2. **预览刷新**：机位视角是固定 POV（不像导演视角可自由环绕），预览缩略图**仅在该相机自身参数变化时按需重渲**，无需每帧。
> 3. **注视目标**：选中某对象后**一次性对准**（填入其中心坐标并 lookAt 一次），之后回到手动坐标。
> 4. **截图数据**：会话内存存储；「发送到画布」保持占位（mock），多选只决定发送哪些。

## B. 机位视角 POV 渲染 + 右侧「摄像机」面板

### B.1 渲染主循环：支持切换渲染相机

最小侵入做法——**渲染相机由 Stage 持有，App 设置**：

```js
// Stage.js
this.activeCamera = null;                 // null=用导演相机
render() { this.renderer.render(this.scene, this.activeCamera || this.camera); }
onResize() { ...; if (this.activeCamera) { this.activeCamera.aspect = W/H; this.activeCamera.updateProjectionMatrix(); } }
```

```js
// App.js
this.activeCameraId = null;
getRenderCamera() { return this.selected?... }          // 见下
setActiveCamera(id) {
  this.activeCameraId = id;
  const ent = this.entities.find(e => e.id===id && e.type==='camera');
  this.stage.activeCamera = ent ? ent.cam : null;
  if (ent) { ent.cam.aspect = 视口宽高比; ent.cam.updateProjectionMatrix(); }
}
```

- `Stage.startLoop` 内部 `this.render()` 已每帧调用，改用 `activeCamera` 即可，**不必把 app 传进 Stage**。
- `capture()` 当前显式传 `camera: this.stage.camera` —— 改为传 `this.stage.activeCamera || this.stage.camera`，截图即按当前渲染相机出图。

### B.2 进入 / 退出机位视角

`App.setCameraView(on)` 扩展（保持方法名/调用点不变）：

- **进入（on=true）**：
  - 选定 active：优先「当前选中的相机」，否则上次 active，否则第一个相机；无相机则提示并回退。
  - `setActiveCamera(activeId)` → Stage 用该 `cam` 渲染。
  - **关闭 OrbitControls**（`rig.controls.enabled=false`）——机位视角是固定 POV，不可环绕（与澄清一致）。
  - 隐藏所有相机可视化（body+helper+相机名牌），尤其 active 自身（复用第一轮 `setCameraGizmoVisible(false)`）。
  - 取景框：按当前 `app.ratio` 显示（`auto` 则全幅）。
  - NavGizmo / 「重置视角」在机位视角隐藏（环绕不可用）。
- **退出（导演视角）**：恢复 `controls.enabled=true`、`stage.activeCamera=null`（回导演相机）、显示相机可视化、显示 NavGizmo。

> viewtabs（顶部「导演视角 / 机位视角」）与 Outliner 点击的衔接：
> - 导演视角下点选相机 → 选中 + 显示摄像机面板 + 显示 gizmo（用于摆放机位）。
> - 机位视角下选相机（经 Outliner 或面板「切换机位」下拉）→ 设为 active + POV 渲染 + 显示面板。
> - 机位视角下相机体已隐藏，无法在 3D 里点选，故切换机位只走 Outliner / 「切换机位」下拉。

### B.3 右侧「摄像机」面板（`Inspector` 扩展）

在 `Inspector.show(entity)` 增加 `entity.type==='camera'` 分支，标题「摄像机」，含两个子 tab：**属性 / 摄像机截图**（复用现有 `.ptabs` 样式）。当选中相机或处于机位视角时展示该面板（`App.select` 已驱动 inspector 显隐）。

#### 属性 tab（对齐截图 1 / 4 / 5）

1. **相机预览框（顶部）**
   - 显示该相机 POV 的渲染缩略图；左上角角标「FOV {n}°」；右下角「全屏扩大」按钮。
   - 刷新（按需）：面板打开、该相机 `position/rotation/注视坐标/fov` 变化时各重渲一次（澄清 2：机位视角固定，无需每帧）。
   - 实现见 B.4 `cameraPreview`。
   - 全屏扩大：打开放大预览 modal（该相机 POV 大图，可叠构图网格）。

2. **名称**：输入框（如「机位5」）→ `app.rename(id, value)`（已有）。

3. **切换机位**：下拉，列出全部相机；选择 → `app.setActiveCamera(id)` + 切换面板到该相机（机位视角下即切 POV）。

4. **位置 X/Y/Z**：数值框，**hover 可左右拖动改值（scrubber）**。复用增强后的 `tripleRow`（见 C.3），绑定 `cam` 所在 `root.position`；变更后刷新预览 + `helper.update()`。

5. **注视目标**：下拉「手动坐标 / <场景中各角色·道具名>」。
   - 选某对象 → 取其世界中心，**一次性**写入「注视坐标」并 `root.lookAt(center)`（再叠加 Dutch roll 若该机位有），随后下拉可保留该名作「上次目标」，但模式回到手动坐标（澄清 3）。
   - 选「手动坐标」→ 用注视坐标手动控制。

6. **注视坐标 X/Y/Z**：scrubber；手动模式下编辑即 `root.lookAt(point)`，刷新预览。

7. **视野角度 (FOV)**：`sliderRow` 20–90° + 数值框 + **ⓘ tooltip**，文案固定为：
   > 控制镜头视野范围。数值越小，画面越近、越聚焦；数值越大，画面越广、能看到更多环境。
   - 变更 → `cam.fov=v; cam.updateProjectionMatrix(); helper.update();` + 刷新预览 + 预览角标。

8. **相机截图区（条件显示）**
   - 标题「相机截图」。
   - 若 `app.shots` 中存在 `cameraId===该相机` 的项，则在面板**最下方**列出其缩略图（名称如「机位5-截图01」）；无则整块不渲染（与「点击下方截屏才出现，否则不显示」一致）。

#### 摄像机截图 tab（对齐截图 2 / 3）

- **全部截图按机位分组**：遍历 `app.shots`，按 `cameraId` 分组，组标题「机位X截图」，组内缩略图 + 名称「机位X-截图0N」。
- **hover 单张**（截图 3）：缩略图上浮遮罩，显示「发送到画布」气泡 + 三个操作按钮：
  - 🗑 删除 → `app.removeShot(id)`
  - ⤴ 发送到画布 → `app.sendShotsToCanvas([id])`（占位）
  - ⤢ 全屏扩大 → 打开放大 modal
- **多选**：缩略图可勾选（点击切换选中 / 角标勾选框），维护一份 `selectedShotIds`。
- **底部固定栏**：
  - 「全部清空」→ `app.clearShots()`（带确认）
  - 「发送到画布」→ 发送**已勾选**项；未勾选时发送全部（或禁用并提示先选择）。**修复现状「一次性把所有截图发过去」的问题**——由多选集决定范围。

### B.4 相机预览渲染（`src/util/cameraPreview.js`，新增）

## C. 截图数据模型 + 控件增强

### C.1 数据模型（`App`）

```js
this.shots = [];          // { id, cameraId, cameraName, name, dataURL, createdAt }
this.selectedShotIds = new Set();
this._shotSeq = {};       // 每机位序号：{ [cameraId]: n }
```

### C.2 方法（`App`）

```js
captureShot()                 // 重构：见下
sendShotsToCanvas(ids?)       // ids 缺省=selectedShotIds 或全部；占位 console.log + toast
removeShot(id)
clearShots()
toggleShotSelected(id)
```

`captureShot()` 重构：

1. 确定归属相机（**每张截图必属于一个真实机位，无「导演视角」孤儿组**）：
   - 机位视角：active CameraEntity。
   - 导演视角且选中了某相机：该相机。
   - **导演视角且无机位上下文（在一个"没有机位的位置"截图）：自动按当前视角新建一个机位**——复用第一轮「当前视角」预设（克隆导演相机 position + OrbitControls.target 的位姿），命名顺延 `机位N`（沿用 `_camCount` 序列），加入 `entities` 并作为本次截图归属相机（同时把它设为选中/active，方便用户随后微调）。
2. 渲染相机 = 上一步确定的相机的 `cam`（保证截图就是该机位所见）。
3. 复用 `capture()` 的 `beforeRender/afterRender` 隐藏 gizmo/环/名牌/**相机可视化**，按 `frameRect` 裁剪 → dataURL。
4. 命名 `${cameraName}-截图0N`，push `shots`，刷新 Inspector（属性页相机截图区 + 截图 tab）+ Outliner（可能新增了机位）。
5. **不再自动下载**（下载改为全屏 modal 内的可选动作）。toast 提示「已截图」（自动建机位时附带「已新建{机位N}」）。

> `latestShot` / `sendLatestShot` 旧逻辑：保留字段或以 `shots` 取代；Dock 已移除 send 按钮，相应接线删除。

### C.3 `tripleRow` 增加 hover 拖动（`widgets.js`）

- 给每个数值输入（或其 `.ax` 标签区）加指针拖动：`pointerdown`→记录起点与初值；`pointermove`→`set(初值 + dx*step*灵敏度)` + `syncAll`；`pointerup`→结束。`cursor: ew-resize`，拖动中不触发文本选中。
- 保留键入。**全局生效**（角色/道具/场景/相机的 XYZ 都获得拖动），低风险、满足「鼠标浮在上面就可以左右拖动」。
- `tripleRow` 需支持每轴 `step` 参数（位置 0.01、注视坐标 0.05 等可调）。

### C.4 可能新增的共享控件（`widgets.js`）

- `dropdownRow({label, options, value, onChange})`：注视目标、切换机位用。
- `infoTip(text)`：FOV 的 ⓘ 提示气泡。
- `thumb`/`shotCard`：缩略图卡片（hover 遮罩 + 操作 + 勾选态），属性页与截图 tab 共用。

## F. 边界与注意事项

- **POV 与 OrbitControls 互斥**：机位视角必须关 orbit，否则环绕会改导演相机但画面来自机位，行为错乱；退出时务必恢复。
- **active 相机 aspect**：进入 POV 与 `onResize` 都要更新 `cam.aspect`；取景框只是裁剪，不改相机投影。
- **截图洁净**：渲染相机预览/截图前隐藏相机体+视锥+名牌+gizmo+选中环，渲染后精确恢复（统一一套钩子，避免遗漏 active 相机自身）。
- **预览开销**：独立小 renderer 惰性创建，仅在参数变化/面板打开时渲染；离开面板可不渲染。注意 WebGL context 数量（移动端上限），可考虑与主 renderer 复用（临时改尺寸再恢复）作为备选。
- **截图归组**：**不存在无 `cameraId` 的截图**——导演视角下在无机位上下文截图会先自动建机位再归属（见 C.2）。命名序号按机位各自递增（`_shotSeq[cameraId]`）。需注意自动建机位会触发 Outliner / 「切换机位」下拉刷新。
- **多选语义**：底部「发送到画布」默认发送勾选项；空勾选时的行为（发全部 or 禁用）需统一，建议「空勾选=禁用并提示」更不易误操作。
- **注视目标一次性**：应用后下拉显示对象名但坐标已落定为手动值；对象之后移动不再跟随（符合澄清，避免每帧 lookAt 开销）。
- **回归**：移除 send 按钮、`captureShot` 不再下载——确认 `selftest` 旧第 7 项（截图比例裁剪）改为从 `shots[last].dataURL` 取图断言尺寸。
