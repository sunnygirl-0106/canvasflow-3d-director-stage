# 技术改动规划 · 第三轮：静态姿势预设替换

> 目标：把右侧「姿势」页里现有的**预设动作**（播放 GLB 内置动画、动态循环）替换为一组
> **静态姿势预设**（站立 / T型 / 行走 / … / 抱臂）。所有预设都是**定格的静止姿态**，
> 通过现有 FK（`character.values` + `applyPose()`）一次性摆好，不再启动 `mixer` 动画。
>
> 交付对象：Claude Code。本文给出范围裁剪理由、数据结构、文件改动、UI 改造、姿势角度初值表、
> 校准流程与验收标准。

---

## 0. 范围裁剪（先看这里）

截图里有 20 个姿势，但当前 FK 引擎**只旋转骨骼**——没有 IK、不位移根节点、没有手指关节、没有道具。
为保证"做出来即可直接用"，**只实现其中 14 个**（双脚/单脚着地的纯旋转姿态），其余 6 个**本轮不做**：

| 不做 | 原因（引擎能力够不到，硬做会穿帮） |
|---|---|
| 坐姿 / 蹲下 / 单膝跪 / 双膝跪 | 需要降低重心。`applyPose()` 只转骨骼、根节点不动且无 IK，脚会穿地或全身悬空。 |
| 倚靠 | 没有可倚靠的墙/道具，纯躯干倾斜看起来像失衡站不稳。 |
| 看手机 | 无手机道具、素体无手指，抓握出不来，只剩"低头抬手"语义含糊。 |

> 这 4 类"重心下沉"姿势 + 倚靠 + 看手机，待日后补 **IK / 根节点落地 / 道具系统** 再加入。
> **实现的 14 个**：站立、T型、行走、跑步、叉腰、鞠躬、思考、格斗、踢球、投掷、推进、招手、伸手、抱臂。

## 1. 背景与现状（必读）

当前「姿势」页由 `src/ui/Inspector.js` 的 `_pose()` 渲染，分两块：

1. **预设动作**：遍历 `ent.clipNames`（GLB 内置动画名，如 `Idle/Walking/...`），点击调用
   `ent.playClip(name)` → `this.mixer.clipAction(clip).play()`，是**动态动画**。还有一个 `T-Pose` 按钮调 `resetPose()`。
2. **姿势调节**：`PoseSliders`，从 `jointConfig.js` 的 `JOINTS` 生成分组滑条，input 写
   `character.values[key]` → `enterManual()` + `applyPose()`。

关键既有能力（直接复用，不要改其语义）：

- `Character.values`：`{ [jointKey]: 角度(度) }`，每个角色各持一份。
- `Character.applyPose()`：`最终四元数 = restQ × Σ setFromAxisAngle(轴, 角度)`，纯 FK 旋转骨骼。
- `Character.enterManual()`：停掉 mixer 动画，置 `poseMode='manual'`，与预设动画互斥。
- `Character.resetPose()`：`values` 清零 + 回静止 + 停动画（静止位 = Mixamo **T-Pose**，手臂水平张开）。
- `JOINTS`（`jointConfig.js`）：27 个关节轴，字段 `{key, label, bone, axis, min, max}`。
- `PoseSliders.syncFromValues()`：外部改了 `values` 后刷新滑条 UI。

> ⚠️ **静止位是 T-Pose，不是自然站立**。所以"站立"需要把手臂从水平放下到体侧；"T型"才是全 0。

**本轮要点**：预设从"播放动画"变成"写一组固定 `values` 再 `applyPose()`"。`playClip/mixer`
相关代码**保留但不再被预设 UI 调用**（避免回归，且模型动画能力日后可能复用）。所有保留的 14 个
姿势都**双脚或单脚着地、靠纯骨骼旋转即可**，不需要位移根节点。

## 3. 新增文件：`src/entities/posePresets.js`

### 3.1 数据结构

每个预设是一组**非零关节角度**（单位：度，键取自 `jointConfig.js` 的 `key`）。未列出的关节默认 0。
所有保留姿势都不需要位移根节点，故**不引入 `rootY` 等下沉字段**。

## 4. `Character.js`：新增 `applyPosePreset()`

在 `Character` 类里加一个方法，统一处理"写 values → 退出动画态 → applyPose"。因为保留的 14 个
姿势都不需要位移根节点，逻辑很简单：

```js
// 顶部 import：
// import { POSE_PRESET_MAP } from './posePresets.js';

/** 应用一个静态姿势预设（静止 FK，不启动动画）。 */
applyPosePreset(presetKey) {
  const preset = POSE_PRESET_MAP[presetKey];
  if (!preset) return;
  // 1) 先清零再写入该预设的非零关节（保证从干净状态叠加）
  for (const j of JOINTS) this.values[j.key] = 0;
  for (const [k, v] of Object.entries(preset.values || {})) {
    if (k in this.values) this.values[k] = v;
  }
  // 2) 退出动画态（停 mixer，置 manual）
  this.enterManual();
  // 3) FK 摆姿
  this.applyPose();
  this.currentPreset = presetKey; // 记录当前选中预设，供 UI 高亮
}
```

> `currentPreset` 初始为 `null`；在 `resetPose()` 里置 `null`。用户**手动拖滑条**时也应取消高亮：
> 在 `PoseSliders` 的 input 回调里加 `ch.currentPreset = null`。
> 注意顺序：`applyPosePreset` 内部先调 `enterManual()`，**之后**再赋 `currentPreset`（如上），
> 避免被 `enterManual` 里的逻辑清掉。

## 5. `Inspector.js`：改造 `_pose()`

把"预设动作（clipNames）"整块替换为姿势预设网格。保留下方滑条，并把段标题改成
**姿势预设 / 姿势调节**，对齐截图。

```js
import { POSE_PRESETS } from '../entities/posePresets.js';

// ---- 姿势页（角色）----
_pose(root, ent) {
  // 1) 姿势预设
  root.appendChild(el('div', { class: 'sec-title', text: '姿势预设', style: { marginTop: '4px' } }));
  const grid = el('div', { class: 'pose-grid' });
  const btns = {};
  const mark = (key) => Object.entries(btns).forEach(([k, b]) => b.classList.toggle('on', k === key));
  for (const p of POSE_PRESETS) {
    const b = el('button', {
      text: p.label,
      class: ent.currentPreset === p.key ? 'on' : '',
      onclick: () => { ent.applyPosePreset(p.key); this.poseSliders?.syncFromValues(); mark(p.key); },
    });
    btns[p.key] = b;
    grid.appendChild(b);
  }
  root.appendChild(grid);

  root.appendChild(el('button', { class: 'minibtn', text: '⟲ 复位姿势', style: { marginTop: '12px' },
    onclick: () => { ent.resetPose(); ent.currentPreset = null; this.poseSliders?.syncFromValues(); mark(null); } }));

  // 2) 姿势调节（滑条）
  root.appendChild(el('div', { class: 'sec-title', text: '姿势调节', style: { marginTop: '16px' } }));
  const host = el('div', {});
  root.appendChild(host);
  this.poseSliders = new PoseSliders(host, ent);
}
```

`_crowdPose()`（群众组）同样加一行预设网格，点击时广播到全部成员：

```js
// 在复位按钮后、host 之前插入预设网格
root.appendChild(el('div', { class: 'sec-title', text: '姿势预设', style: { marginTop: '4px' } }));
const grid = el('div', { class: 'pose-grid' });
for (const p of POSE_PRESETS) {
  grid.appendChild(el('button', { text: p.label, onclick: () => {
    crowd.members.forEach((m) => m.applyPosePreset(p.key));
    this.poseSliders?.syncFromValues();
  }}));
}
root.appendChild(grid);
```

> 删除/不再渲染原 `clips`（clipNames 循环）与 `T-Pose` 按钮、`该模型无内置动画` 提示。
> `T型` 预设已等价覆盖原 T-Pose 功能。`playClip` 等方法保留在 `Character.js` 中不动。

## 6. 关节角度校准流程（重要，参照既有 §5.4）

`posePresets.js` 中的角度是**起始估值**。各骨头本地坐标轴方向不完全一致，可能出现：
方向相反、绕错轴、左右不对称。请按下述流程逐个核对（建议加载 `Xbot.glb` 作基准）：

1. `npm run dev`，加一个角色，进「姿势」页，逐个点预设观察。
2. 若某关节**反向**：把该 key 的角度取负即可（无需改模型）。
3. 若某关节**绕错轴**（如该动应抬腿却变成外展）：说明该骨头此动作对应的轴与 `jointConfig`
   里写的 `axis` 不符——优先在 `jointConfig.js` 校准 `axis`（会同时修正滑条语义），再回填预设。
4. 左右成对务必镜像：`lArmAbd` 与 `rArmAbd`、`lFore` 与 `rFore` 等通常**符号相反**。
5. 改完角度后再点一次预设确认；预设是纯静态，不应有任何抖动/循环。

校准只动 `posePresets.js` 的数值（必要时 `jointConfig.js` 的 `axis`），不动 FK 框架。

## 7. 已知限制 / 取舍

- **本轮只做 14 个**：坐/蹲/单膝跪/双膝跪/倚靠/看手机已剔除（见 §0），因为现引擎无 IK / 不位移
  根节点 / 无手指 / 无道具，硬做会穿帮。
- **保留姿势全部脚着地**：行走/跑步/踢球只是抬起一条腿（合法旋转），支撑腿与地面关系不变，
  无悬空/穿地问题，故不需要根节点位移。
- **预设是定格**：行走/跑步/踢球/投掷等取一个有代表性的"定格帧"，非动画。动态动画能力仍由
  `playClip/mixer` 保留，本轮 UI 不暴露。
- **角度初值需校准**：见 §6，首次合并后必做。

## 8. 验收标准

1. 角色「姿势」页顶部显示 **姿势预设** 网格（4 列），文案与顺序为 §3.1 的 14 项。
2. 点击任一预设：角色立即定格到对应静止姿态，按钮高亮，下方 **姿势调节** 滑条同步到该预设角度。
3. 拖动任一滑条后，预设高亮取消（回到自由微调态）。
4. 「⟲ 复位姿势」回到 T-Pose（全 0）并取消高亮。
5. 任一预设下角色脚不穿地、不悬空（保留姿势本就脚着地；经 §6 校准方向后）。
6. 全程无控制台报错；预设为静止，无动画抖动/循环。
7. 群众组（crowd）选中时，预设网格点击同步作用到全部成员。
8. `npm run build` 通过；`selftest.mjs` 新增用例通过。
